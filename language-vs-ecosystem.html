Language vs Ecosystem

Nyra is a general-purpose language. Frameworks and packages complete it for each domain — the compiler does not bake in backend-only stacks.

Two layers

┌─────────────────────────────────────────────────────────┐
│  Layer 2 — Community / NyraPkg                          │
│  ny-sqlite, ny-redis, game engines, CLI libs, …         │
├─────────────────────────────────────────────────────────┤
│  Layer 1 — Nyra language + strong stdlib (Core)         │
│  ownership, Result/Option/?, crypto, db, serde, net     │
└─────────────────────────────────────────────────────────┘

Layer 1 — Language + stdlib

What ships with the compiler and belongs in every project:

Strong batteries-included stdlib

Crypto, SQLite, WebSocket, compression, and full serialization are stdlib domains — native code in stdlib/rt/. Some modules are still stubs while implementations land; import paths stay stable. See stdlib → Implementation status · stdlib/README.md.

  • Core language: ownership, borrow check, enums, Option<T> / Result<T,E> (with import), monomorph generics
  • Stdlib (shipped): collections, strings, FS, TCP, net/http client + server, random_bytes, builtins (print, …)
  • Stdlib (in progress): SHA/AES, SQLite, WebSocket, gzip, full serde — stable paths, native implementations landing in-tree
  • Toolchain (batteries-included): nyra build/run/check/test/fmt/pkg — see Toolchain

CI gate: bash scripts/test-all.sh — includes CONF-LANG (scripts/conformance-tests.sh), Rust CONF-*, compiletest suite (~1.6k fast profile), and tests/nyra/ smoke. See Toolchain → Testing layers.

Layer 2 — NyraPkg & community

Database drivers, crypto, compression, domain SDKs — published via nyra pkg, not the compiler:

nyra pkg install ny-sqlite@^0.1.0
import "pkg/ny-sqlite"

See NyraPkg. Anyone can publish; the language stays neutral.

net/http

HTTP client, server, router slots, and handler dispatch live in stdlib (stdlib/net/http/). See net/http reference and examples/net_http_smoke.ny.

Who enters a domain?

The language is ready for any field (systems, web, embedded, games, data). Teams and community bring Nyra into a domain via packages and app modules — not by growing the compiler/stdlib for one vertical.
