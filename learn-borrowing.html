Nyra Borrowing

Borrowing lets you use a value without taking ownership. References (& and &mut) are checked at compile time so you never get dangling pointers or data races.

Auto-borrow at calls (v2.8)

You usually do not need to write & at every call. If the function expects a borrow, Nyra inserts it:

fn save(u) {
    print(u.name)
}

fn main() {
    let user = { name: "Ada", age: 30 }
    save(user)           // compiler rewrites to save(&user)
    print(user.name)     // OK — not moved
}
struct User {
    name: string
    age: i32
}

fn save(u: &User) {
    print(u.name)
}

fn main() -> void {
    let user: User = User { name: "Ada", age: 30 }
    save(user)           // compiler rewrites to save(&user)
    print(user.name)     // OK — not moved
}

Use save(move user) for an explicit move or save(clone user) to duplicate. See Ownership UX.

Immutable borrow: &T

Read-only access — many readers allowed, no writers:

fn len(s) {
    return strlen(s)
}

fn main() {
    let msg = "hello"
    let n = len(&msg)      // borrow — msg still owned by main
    print(n)
    print(msg)              // still valid
}
fn len(s: &string) -> i32 {
    return strlen(s)
}

fn main() -> void {
    let msg: string = "hello"
    let n = len(&msg)      // borrow — msg still owned by main
    print(n)
    print(msg)              // still valid
}

Mutable borrow: &mut T

Exclusive write access — only one &mut at a time:

fn main() {
    let mut count = 0
    let r = &mut count
    // cannot use count directly while r is active
    count = count + 1       // through mut binding OK if no conflicting borrow
}
fn main() -> void {
    let mut count: i32 = 0
    let r = &mut count
    // cannot use count directly while r is active
    count = count + 1       // through mut binding OK if no conflicting borrow
}

Non-lexical lifetimes (NLL)

Borrows end at the last use of the reference, not at the closing }. This lets you write natural code:

fn main() {
    let mut s = "hello"
    let r = &s
    print(r)                 // last use of r — borrow ends HERE
    s = "world"              // OK now
}
fn main() -> void {
    let mut s = "hello"
    let r = &s
    print(r)                 // last use of r — borrow ends HERE
    s = "world"              // OK now
}

Without NLL you would have to introduce extra blocks just to end borrows early.

Rules (the borrow checker)

  1. Any number of &T OR exactly one &mut T — never both at once.
  2. References must not outlive the data they point to.
  3. You cannot move a value while it is borrowed.
// ERROR: cannot borrow as mutable while immutable borrow active
fn bad() {
    let mut s = "x"
    let a = &s
    let b = &mut s
    print(a)
}
// ERROR: cannot borrow as mutable while immutable borrow active
fn bad() {
    let mut s = "x"
    let a = &s
    let b = &mut s
    print(a)
}

Borrowing vs moving

SyntaxOwnershipWhen to use
foo(s)Move (for string)Callee takes full ownership
foo(user)Auto-borrow (if param is &User)Compiler inserts & — binding kept
foo(&s)BorrowExplicit borrow (same as auto-borrow)
foo(&mut s)Exclusive borrowCallee may modify through the reference

References and auto-drop

Borrowing does not delay cleanup forever. When the owner binding goes out of scope, the owned value is still dropped — but only if it wasn't moved. References themselves are Copy and hold no heap resources.
