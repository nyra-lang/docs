Nyra Data Types

Built-in types: integers, booleans, strings, and more.

All types are optional. Nyra infers variable types, function parameter types, return types, struct shapes, and generic call sites at compile time — annotations are never required unless inference fails. Types are erased before run time: zero memory or CPU cost vs writing them out.
CategoryOptional?Example (without types)
VariablesYeslet score = 100
Function paramsYesfn add(a, b) { return a + b }
Return typesYesfn twice(x) { return x + x }
Structs / objectsYeslet u = User("Ada")
GenericsYesid(42) — T inferred at call site
Split listsYeslet parts = "a,b".split(",") — annotate VecStr when you want

Scalar types

Integer types are optional — Nyra infers i32 for literals like 42. Annotate only when you need a specific width or signedness (same as Rust).

TypeExampleNotes
i8, i16, i32, i64, i128let n: i32 = 42Signed integers; i32 is the default for int literals
u8, u16, u32, u64, u128let b: u8 = 255Unsigned integers — bytes, flags, buffers
isize, usizelet len: usize = 0Pointer-sized signed / unsigned (64-bit on most targets)
f321.5f32, 0.532-bit float — games, graphics, GPU; suffix f32 or annotate f32
f643.14, 19.9964-bit float (IEEE-754 double) — default for float literals; prices, science
char'a', '\n'Unicode scalar (for-in over strings yields char)
booltrue, falseBoolean
string"hello"UTF-8 text (heap-owned when dynamic)
void—No return value (Rust () unit type equivalent)

Annotate explicitly when you want: let age: i32 = 25, let price: f64 = 9.99, let parts: VecStr = line.split(","). Mixed math promotes integers to f64 when needed (1 + 2.5). Full catalog: Types & data · Language reference → Type names.
