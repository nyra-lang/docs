Dungeon Steps

A small but complete multi-file app in the Nyra repository. Use it as a map from “I read the syntax page” to “I can structure a real project.”

Overview reference app

Dungeon Steps is a deterministic grid mini-game: the player moves on a 4×4 map, collects gold, avoids traps, and can win by reaching the goal tile. The simulation runs automatically in run_dungeon(); main.ny prints a few demo values so you can see the engine without playing interactively.

Source lives at App/NyraApp/ in the Nyra repository. It is intentionally larger than the calculator example so you can study imports, shared types, and game logic split by concern.

Run the app

git clone git@github.com:nyra-lang/nyra.git
cd Nyra/App/NyraApp
nyra run .
Project root only Run from the directory that contains main.ny. Use nyra run . — not nyra run main.ny — so imports resolve. See Modules and Imports guide.

After building Nyra from source locally:

./scripts/dev-install.sh   # from repo root
cd App/NyraApp
nyra run .

Expected output

Dungeon Steps
65
0
3
LineFromMeaning
Dungeon StepsAPP_TITLE constApp title string
65run_dungeon()Final score after auto-play (e.g. one gold +15 and goal +50)
0outcome_code(g1)STATUS_PLAYING after one eastward move from a fresh game

Scores can change if you edit the map or MAX_TURNS; the last two lines stay 0 and 3 unless you change main.ny.

Project layout

App/NyraApp/ main.ny ← entry, prints demo output nyra.mod src/ config.ny ← file-level const types.ny ← enum Tile, Dir; struct GameState world.ny ← map, clamp, turn % 4, directions engine.ny ← rules, simulation, tests

main.ny only imports what it needs for the demo. engine.ny imports world.ny, which imports types.ny and config.ny — a typical layered dependency tree.

How the pieces connect

  main.ny
     │  import config, engine, types
     ▼
  engine.ny ──► world.ny ──► types.ny
     │              │            config.ny
     │              └── tile_at, clamp_axis, dir_for_turn
     └── new_game, step_turn, run_dungeon, match on Tile/Dir

config holds tuning constants (grid size, damage, status codes). types defines shared data shapes used everywhere. world is pure geography and movement helpers. engine applies tile effects and runs the turn loop.

Built-in tests

engine.ny includes small tests you can run from the project root:

nyra test .

Examples: map_has_gold_tile, fresh_game_is_playing, turn_phase_uses_mod.

Use tests as a template when you add your own modules — keep checks next to the logic they verify.

Suggested learning path

  1. Run nyra run . and match each output line to main.ny (see Expected output).
  2. Read src/config.ny and src/types.ny — no imports to untangle yet.
  3. Trace one move: move_player → advance_parts → tile_at in world.ny.
  4. Read run_dungeon and follow step_turn / dir_for_turn for auto-play.
  5. Change GOLD_VALUE or trap position in world.ny, rebuild, and compare the printed score.
  6. Run nyra test . and add a test fn for a rule you care about.
