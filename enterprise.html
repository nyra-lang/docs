Workspaces & large teams

Nyra supports multi-module workspaces (nyra.mod), a language conformance suite (CONF-WS-*), and documented ownership patterns for teams — without requiring a service mesh or distributed tracing stack inside the language.

Evaluation status

AreaStatusNotes
nyra.mod workspaces✅Multi-module layout, stdlib requires, per-service entrypoints
Workspace docs✅This page and Toolchain → conformance
Conformance suite✅CONF-WS-001 … 004 in conformance/workspace.rs
Monolith structs + strings✅Composite drop, partial moves (examples/monolith_struct_smoke.ny)
Shared ownership / graphs✅Arc<T> (examples/graph_arc_smoke.ny)
Async bootstrap✅spawn + await ready-probe pattern (CONF-WS-002)
Distributed tracing / service meshExternalIntegrate via FFI or sidecar; not part of Nyra stdlib

Workspace layout (nyra.mod)

Declare module identity and stdlib dependencies once per project:

module my.api

require stdlib.net.http
require stdlib.arc
require stdlib.option

Typical tree for a service with shared domain types:

my_api/
  nyra.mod
  main.ny
  lib/common/          # domain types + config
  handlers/            # HTTP-facing handlers
  workers/             # background jobs, Arc-shared state

Smoke examples: examples/graph_arc_smoke.ny, examples/monolith_struct_smoke.ny, examples/net_http_smoke.ny.

Async ready probe

Services can bootstrap async work with promises and expose readiness via Option<i32> (CONF-WS-002). See Async guide for spawn/await semantics.

Ownership review checklist

AreaReview focus
Strings in structsComposite auto-drop; extract fields before second use
Shared graphsArc<string> / Arc<i32>; last drop frees payload
Cross-module typesOne consuming call per moved struct (tenant summary OR display id)
Async tasksSpawn captures Send; await before reusing moved handles
GenericsMonomorph instances in borrowck (Option__T, Arc__T)

Conformance suite (CONF-WS)

IDChecks
CONF-WS-001Cross-module struct + string composite drop
CONF-WS-002Option<i32> async ready-probe pattern
CONF-WS-003examples/graph_arc_smoke.ny
CONF-WS-004examples/monolith_struct_smoke.ny

Run locally:

cargo test -p compiler --test conformance conf_ws

CI matrix (recommended)

  • cargo test -p compiler --test conformance conf_ws conf_arc conf_own
  • Per-project nyra check on each nyra.mod workspace
  • make test-all for full gate

Explicitly out of scope

These belong in platform tooling around Nyra, not in the compiler/stdlib MVP:

  • OpenTelemetry / distributed tracing exporters
  • Service mesh (Istio, Linkerd, etc.)
  • Fully async HTTP accept loop (roadmap; sync loop today)
