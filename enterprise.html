Enterprise & large teams

Nyra v2.7 delivers production-shaped workspaces, a conformance suite (CONF-ENT-*), and documented ownership patterns for teams of 50+ developers — without requiring a service mesh or distributed tracing stack inside the language.

Evaluation status (v2.7)

AreaStatusNotes
nyra.mod workspaces✅Multi-module layout, stdlib requires, per-service entrypoints
Enterprise docs✅This page, README in example project, docs/conformance/enterprise.md
Conformance suite✅CONF-ENT-001 … 006 + scripts/enterprise-check.sh
Monolith structs + strings✅v2.4 composite drop, partial moves
Shared ownership / graphs✅v2.5 Arc<T>
Async microservices✅v2.6 spawn + await bootstrap, HTTP health
Distributed tracing / service meshExternalIntegrate via FFI or sidecar; not part of Nyra stdlib

Workspace layout (nyra.mod)

Declare module identity and stdlib dependencies once per service:

module enterprise.platform

require stdlib.enterprise
require stdlib.microservice
require stdlib.arc
require stdlib.option

Recommended tree (see examples/projects/enterprise_platform/):

enterprise_platform/
  nyra.mod
  main.ny
  lib/common/          # domain types + config
  services/api/        # HTTP-facing handlers
  services/worker/     # background jobs, Arc-shared labels

Minimal team service: examples/projects/team_api/ (single entry, enterprise bootstrap + health JSON).

stdlib/enterprise

import "stdlib/enterprise/mod.ny"

fn main() {
    let ready = Enterprise_bootstrap()       // async spawn + await
    let meta = EnterpriseMeta_new("api", "2.7.0", "prod")
    let health = Enterprise_health_json(meta)
    let probe = Enterprise_ready_probe()
}
import "stdlib/enterprise/mod.ny"

fn main() -> void {
    let ready = Enterprise_bootstrap()       // async spawn + await
    let meta = EnterpriseMeta_new("api", "2.7.0", "prod")
    let health = Enterprise_health_json(meta)
    let probe = Enterprise_ready_probe()
}
  • EnterpriseMeta — owned strings with composite auto-drop
  • Enterprise_health_json — builds JSON health payload (move fields out of meta)
  • Enterprise_ready_probe — returns Option<i32> after bootstrap

Async microservices

import "stdlib/microservice.ny"

fn main() {
    let cfg = ServiceConfig_new("orders-api", "0.0.0.0", 8080)
    let ready = Service_bootstrap_async()
    print(ready)
    // Service_run_async_health(cfg)   // blocking HTTP listen
}
import "stdlib/microservice.ny"

fn main() -> void {
    let cfg = ServiceConfig_new("orders-api", "0.0.0.0", 8080)
    let ready = Service_bootstrap_async()
    print(ready)
    // Service_run_async_health(cfg)   // blocking HTTP listen
}
  • Send — spawn captures must be Send
  • Ownership — move payloads into spawn; no &mut across await
  • HTTP — sync accept loop in stdlib/net/http; async work via promises

Ownership review checklist

AreaReview focus
Strings in structsComposite auto-drop; extract fields before second use
Shared graphsArc<string> / Arc<i32>; last drop frees payload
Cross-module typesOne consuming call per moved struct (tenant summary OR display id)
Async tasksSpawn captures Send; await before reusing moved handles
GenericsMonomorph instances in borrowck (Option__T, Arc__T)

Run locally:

bash scripts/enterprise-check.sh

Spec: docs/conformance/enterprise.md

CI matrix (recommended)

  • bash scripts/enterprise-check.sh
  • cargo test -p compiler --test conformance conf_ent conf_msvc conf_arc conf_own
  • Per-service nyra check on each nyra.mod project

Explicitly out of scope

These belong in platform tooling around Nyra, not in the compiler/stdlib MVP:

  • OpenTelemetry / distributed tracing exporters
  • Service mesh (Istio, Linkerd, etc.)
  • Fully async HTTP accept loop (roadmap; sync loop today)
