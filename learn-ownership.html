Nyra Ownership

Every value in Nyra has exactly one owner. When the owner goes away, the value is cleaned up automatically — no garbage collector required.

Why ownership?

In languages with manual memory management (C), forgetting free causes memory leaks. In languages with garbage collection (Java, Go), you pay runtime overhead and unpredictable pauses.

Nyra picks a third path: the compiler tracks who owns each value and inserts cleanup code before your program ever runs.

The two kinds of values

Copy types — cheap duplicates

Numbers, booleans, f64, and structs whose fields are all Copy (like Point) duplicate on assign:

let x = 42
let y = x       // y gets a copy of 42
print(x)        // still works — x was not stolen
print(y)

let p1 = { x: 1, y: 2 }
let p2 = p1     // auto-Copy — both usable (v3.0)
print(p1.x)
let x: i32 = 42
let y = x       // y gets a copy of 42
print(x)        // still works — x was not stolen
print(y)

let p1 = { x: 1, y: 2 }
let p2 = p1     // auto-Copy — both usable (v3.0)
print(p1.x)

Move types — one owner only

Strings in Nyra live on the heap. When you assign or pass a string, ownership moves to the new binding:

let greeting = "Hello"
let copy = greeting   // MOVE — greeting is now invalid
print(copy)           // OK
// print(greeting)     // COMPILE ERROR: use of moved value
let greeting: string = "Hello"
let copy = greeting   // MOVE — greeting is now invalid
print(copy)           // OK
// print(greeting)     // COMPILE ERROR: use of moved value

This prevents two parts of your program from both thinking they should free the same memory.

Who cleans up?

When a binding goes out of scope, Nyra automatically frees heap memory:

fn main() {
    let name = "Nyra"     // heap string allocated
    print(name)
}                         // compiler inserts free(name) here
fn main() -> void {
    let name = "Nyra"     // heap string allocated
    print(name)
}                         // compiler inserts free(name) here

You never write free in normal Nyra code. If your program compiles, every owned string gets freed exactly once.

Moving into functions

fn show(msg) {
    print(msg)
}                       // show() frees msg when it returns

fn main() {
    let text = "Hi"
    show(text)          // text moved into show — main no longer owns it
}
fn show(msg: string) {
    print(msg)
}                       // show() frees msg when it returns

fn main() -> void {
    let text: string = "Hi"
    show(text)          // text moved into show — main no longer owns it
}

When the callee takes &T instead of T, Nyra auto-borrows — you write save(user) and the compiler passes &user. See Borrowing.

Functions that create heap values

extern fn strcat(a, b){
    let hello = strcat("Hello, ", "world")
    print(hello)
}                       // hello freed automatically
extern fn strcat(a, b){
    let hello = strcat("Hello, ", "world")
    print(hello)
}                       // hello freed automatically

The return value of strcat is a new owned string. You are responsible for it from the moment it returns — but cleanup is still automatic at scope end.

What you cannot do

  • Use a value after moving it
  • Free the same string twice (compiler prevents this)
  • Return a reference to a local variable (see Borrowing)
